# Safety contract — repoops v0

## Core rule

`repoops` is a metadata reporter. It is not a repository repair tool.

## Forbidden Git commands

The implementation must not run:

```text
git pull
git push
git reset
git clean
git checkout
git switch
git add
git commit
git stash
```

## Allowed Git commands

The implementation may run only:

```text
git rev-parse --is-inside-work-tree
git rev-parse --abbrev-ref HEAD
git rev-parse --short HEAD
git status --porcelain=v1
```

## Secret handling

The scanner may see changed file paths from Git status. It must not read file contents.

Secret-like path indicators include:

```text
.env
secret
secrets
credential
credentials
token
key
private
pem
id_rsa
id_ed25519
```

If a changed path is secret-like:

1. count it normally;
2. add `possible_secret_file`;
3. redact it before rendering or notifying.

## Redaction examples

```text
.env -> [REDACTED_SECRET_PATH]
config/prod.credentials.json -> config/[REDACTED_SECRET_PATH]
secrets/api_token.txt -> [REDACTED_SECRET_PATH]/[REDACTED_SECRET_PATH]
```

## Notification rule

Notification backends must receive only the Markdown report generated by `repoops.report`, never raw Git output, full diffs, environment values, or file contents.
